# ST-12 · Бот-команда `/costs [місяць]`

**Тип:** покращення (BA-аналіз 2026-07-21) · **Оцінка:** S · **Пріоритет:** низький
(зручність; дані вже є) · **Залежності:** — (`report_logs` + індекс
`(user_id, created_at)` ✅)

## Проблема

Витрати на Claude — наскрізна тема проєкту (`PRICES`, `cost_usd` у кожному
`ReportLog`, cost-safety-правило в CLAUDE.md), але подивитись «скільки я спалив
цього місяця» можна лише у веб-браузері таблиць (`/me/report_logs`) — рядок за
рядком, без сум.

## Acceptance criteria

- [ ] `/costs` (і `/costs 2026-06` для конкретного місяця): одна агрегація по
      `report_logs` поточного юзера — сума $ за місяць, розбивка по `kind`
      (report/plan/digest/ask/…), кількість викликів, cache-hit-share
      (`cached=True` — безплатні), топ-3 найдорожчі виклики (kind + дата + $).
- [ ] Чистий DB-рід: без Garmin, без Claude, без `user_runtime`
      (`_resolve_user`-патерн `/records`/`/compare`).
- [ ] Порожній місяць → дружнє «викликів не було».
- [ ] Тест у стилі наявних хендлер-тестів: агрегати правильні, чужі рядки не
      враховані.

## Підводні камені

- `cached=True`-рядки мають `cost_usd` ≈ 0 — рахувати їх у кількість (показує
  ефективність кешу), але не в суму; показати hit-rate явно, це найкраща
  реклама PERF-02.
- Місяць парсити терпимо (`YYYY-MM`; сміття → підказка формату), межі місяця — по
  `created_at` у локальній таймзоні джоб (Europe/Warsaw), як решта бот-логіки.
