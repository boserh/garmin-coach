# NF-11 · Гідрація/харчування для ключових сесій у спеку

**Тип:** нова фіча (BA-аналіз 2026-07-21) · **Оцінка:** S–M · **Пріоритет:** середній,
**сезонно — зараз** (липнева спека — той самий моментум, під який робився EP-13) ·
**Залежності:** EP-13 ✅ (`fetch_forecast`/hourly-слоти), ST-01 ✅ (`plan_today`)

## Суть

Погода вже двигає сесії (EP-13), але не каже, *як пережити* довгу/темпову в +30°:
скільки пити, чи брати гелі, коли стартувати. Порада «саме на сьогодні», якої немає
в Garmin: детермінований калькулятор + один абзац у ранковому DM, коли сьогоднішня
сесія важка (`ADAPT_HEAVY_TYPES`/long) і `feels_max_c` вище порога.

## Дизайн

- `app/fueling.py` — чистий Python (нуль LLM): вхід — тривалість сесії (з
  `steps`/дистанції та pace-діапазону) + feels-like на годину сесії (hourly-слоти
  вже у forecast) → `{fluid_ml_h, carbs_g_h, notes[]}` за консервативними
  спортивними нормами (напр. >60–75 хв → вода обов'язково; >90 хв → 30–60 г/год
  вуглеводів; спека → +електроліти, підказка про ранковий слот з мінімальним
  feels-like).
- Подача — **без нового Claude-виклику**: компактний `fueling`-знімок у контекст
  ранкового звіту (`run_analysis` → `analyze_with_stats` `user_content["fueling"]`
  **і в `_cache_key`** — наскрізна пастка) + секція в `SYSTEM`: озвучити цифри, не
  перераховувати їх.
- Тихо, коли неактуально: нема плану на сьогодні / сесія легка / прохолодно /
  нема локації → ключа `fueling` в контексті просто немає (патерн `norm`/`records`).

## Acceptance criteria

- [ ] `fueling.advise(session, forecast)` покритий тестами: короткa легка → `None`;
      довга в спеку → рідина+вуглеводи+слот; довга в прохолоду → лише тривалісні
      поради; відсутній форкаст → `None` (best-effort, як увесь weather-шлях).
- [ ] Знімок у `_cache_key` (інакше кеш віддасть учорашній звіт без порад).
- [ ] Оцінка тривалості сесії з `steps`/`dist`+pace — з фолбеком на грубу оцінку
      по типу, коли структури немає.
- [ ] Нуль нових `ReportLog`-kind'ів і нуль нових викликів (їде всередині
      наявного ранкового Sonnet-виклику).

## Підводні камені

- Не медичні поради: загальні спортивні орієнтири, у `SYSTEM` — консервативний тон
  (той самий принцип, що `SYSTEM_INJURY`/`SYSTEM_HEALTH`).
- Правило близькості з ST-03/weather: поради лише коли сесія **сьогодні** — не
  засмічувати звіт гелевими розрахунками на п'ятницю.
- Пороги (температура, тривалість) — у `Settings` з дефолтами, як
  `WEATHER_HEAT_FEELS_C`.

## Вартість

Нуль додаткових викликів — контекст наявного ранкового звіту.
