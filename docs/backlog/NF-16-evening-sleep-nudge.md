# NF-16 · Вечірній sleep-nudge перед важкою сесією

**Тип:** нова фіча (BA-аналіз 2026-07-21, друга партія) · **Оцінка:** S–M ·
**Пріоритет:** середній (єдина точка продукту, що впливає на відновлення *до* того,
як ніч втрачена) · **Залежності:** NF-01 ✅ (коридор `sleep_h`), план ✅; синергія
з ST-14 (вечір — поняття таймзони)

## Суть

Весь продукт реагує вранці, коли ніч уже втрачено. А дані для превентивної дії
лежать у БД ще звечора: завтрашня сесія (план), накопичений недосип
(`extra.sleep_need`/`sleep_feedback`, NF-01-коридор по `sleep_h`). Один короткий
вечірній DM «завтра інтервали; останні ночі нижче твоєї норми — сьогодні варто
лягти до ~22:30» — детермінований, тихий за замовчуванням.

## Дизайн

- Легка вечірня джоба `sleep_nudge_job` (`run_daily` о `SLEEP_NUDGE_HOUR`, дефолт
  21:00 Europe/Warsaw — патерн `weather_plan_job`; через `for_each_user`).
- Умова (чистий Python, `app/sleepnudge.py` або функція в `subjective`-стилі):
  завтра важка сесія (`ADAPT_HEAVY_TYPES`/long з активного плану) **і** сигнал
  недосипу — `sleep_h` нижче NF-01-коридору ≥2 з останніх 3 ночей, або Garmin
  `sleep_need` помітно вище фактичного сну. Немає обох умов → тиша («нуль
  конфлікту ⇒ нуль повідомлень», правило EP-13).
- Час відбою = цільова тривалість сну (Garmin sleep need, фолбек — p50 коридору
  +30 хв) відрахована від типового часу підйому (медіана wake-time з останніх
  ночей у sleep-даних; немає — не називати час, лише «лягай раніше»).
- Guard: раз/вечір (`bot_state` `sleep_nudge:<date>`), тумблер — реюз
  `User.alerts_enabled` (це wellness-пуш, той самий клас, що EP-08) +
  процесний `SLEEP_NUDGE` on/off.
- Детермінований текст, нуль LLM.

## Acceptance criteria

- [ ] Нудж лише при (важка сесія завтра) ∧ (сигнал недосипу); кожна умова окремо
      → тиша. Тести на обидві півумови + повний кейс.
- [ ] Раз/вечір; `alerts_enabled=False` або `SLEEP_NUDGE=False` → тиша;
      деактивований/без chat_id → скіп (наявні guard'и `for_each_user`).
- [ ] Чистий DB-рід (`load_credentials`-патерн, без Garmin/MFA — дані вже
      синкнуті вранці).
- [ ] Wake-time медіана рахується з наявних sleep-даних (`extra`), відсутність →
      текст без конкретного часу.
- [ ] Нуль Claude-викликів.

## Підводні камені

- Не стати будильником-нянькою: без сигналу недосипу перед важкою сесією — тиша
  (не «перед кожним темповиком»); частота природно обмежена перетином двох умов.
- Вечір — поняття таймзони: до ST-14 година глобальна (Europe/Warsaw), у тікеті
  ST-14 цей нудж додати до списку per-user-перевірок.
- Дані про сон учорашньої ночі — найсвіжіше, що є ввечері (сьогоднішня ніч ще не
  відбулась) — формулювання «останні ночі», не «сьогодні».

## Вартість

Нуль LLM; нуль Garmin-викликів (чистий DB-рід).
