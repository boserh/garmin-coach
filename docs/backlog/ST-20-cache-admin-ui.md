# ST-20 · Інвалідація кешів з UI (адмін-сторінка /admin/cache)

**Тип:** покращення (BA-аналіз 2026-07-23, кат. A — керування даними) · **Оцінка:** S ·
**Пріоритет:** низький · **Поверхня:** веб (admin-only) · **Залежності:** — (OPS-06
наповнює цю ж сторінку метриками)

## Проблема

Два кеші живуть без жодного керування з UI: `llm_cache` (таблиця, purge лише лінивий
на write) і дисковий `GARMIN_CACHE_DIR` (файл на кожен `series:`/`splits:`/`exercise:`/
`workout:`-ключ, TTL до 365 днів, росте на SD-картці роками). Скинути конкретний ключ
або прострочені — лише ssh + `rm`/SQL. В `/ui` адмін уже може видаляти рядки
`bot_state` (`POST /ui/bot_state/delete`) — кеші лишились сліпою зоною.

## User story

Як адмін я хочу сторінку стану кешів з кнопками очистки, щоб чинити биті/застарілі
записи без ssh до Pi.

## Обсяг

**Входить:** `GET /admin/cache`: зведення llm_cache (рядків усього/прострочених,
розмір) + garmin-кешу (файлів/байтів за префіксами); дії: «purge прострочених
llm_cache», «видалити llm_cache повністю» (з confirm і попередженням «наступні
виклики — платні»), «видалити garmin-кеш активності <activity_id>» (усі ключі цієї
активності), «purge прострочених файлів garmin-кешу».

**Не входить:** per-key браузер вмісту кешу (зайве); не-адмінський доступ
(інвалідація впливає на всіх юзерів процесу).

## Acceptance criteria

- [ ] Сторінка під `require_admin`; усі дії — POST з confirm.
- [ ] Purge прострочених у llm_cache видаляє лише `expires_at < now`; повна очистка
      показує, скільки видалено.
- [ ] Видалення по activity_id зносить `series:v2:`/`splits:v1:`/`exercise:v2:`
      файли саме цієї активності й нічого більше.
- [ ] Тести: не-адмін → 403; purge рахує правильно; чужі файли не зачеплені.

## Технічні дотики

- `app/db/llm_cache.py`: `purge_expired()`/`purge_all()`/`stats()`.
- `app/garmin/client.py`: `cache_stats()`/`cache_del_activity(aid)`/
  `cache_purge_expired()` (обхід каталогу, парс TTL з тих самих констант).
- `app/routers/admin.py` + новий шаблон `cache.html`.

## Вартість LLM / запити до Garmin

Прямо 0/0. Побічно: повна очистка llm_cache робить наступні повторні виклики
платними — тому confirm; це усвідомлений адмін-інструмент.

## Ризики

- Видалення файлів під конкурентним читанням — atomic-replace семантика вже є
  (per-key файли), del окремого файлу безпечний.
