# NF-08 · Особистий MCP-сервер над БД (експеримент)

**Тип:** нова фіча (аудит 2026-07) · **Оцінка:** M (~1.5 тижні) · **Пріоритет:**
експеримент (RICE 0.9; головний диференціатор репозиторію для
open-source-стратегії — ANALYSIS.md §4.3) · **Залежності:** EP-09 (ті самі
read-хелпери — подвійне використання одного коду)

## Суть

Тренд «дані назад користувачу» (пор. garmin-givemydata); наша БД глибша.
Тонкий **read-only** MCP-шар поверх user-scoped read-хелперів, які робитиме
EP-09 (`query_activities`/`query_daily`/`aggregate_weekly`) — «поговорити зі
своїми даними» з Claude Desktop/Code без нашого UI. Для власника-розробника —
потужний ad-hoc аналіз. Головний ризик — scope creep: тримати read-only.
