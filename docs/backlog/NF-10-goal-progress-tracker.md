# NF-10 · `/goal` — кількісний прогрес до цілі з проєкцією

**Тип:** нова фіча (BA-аналіз 2026-07-21) · **Оцінка:** M · **Пріоритет:** середній
(відповідь на головне питання будь-якого плану — «чи я встигаю») · **Залежності:**
дані вже в БД (`extra`: race predictions, VO2max; глибока історія після
import-export); синергія з EP-05 фазою 0 (типізація `target_date`)

## Суть

Дайджест каже «відстаєш» якісно, але ніде немає числа: «прогноз 10K зараз 52:30,
ціль 50:00, до старту 6 тижнів, темп покращення −20 с/тиждень → на треку /
відстаєш на ~X с». Усі складові вже зберігаються щодня в `daily_metrics.extra`
(race predictions по 4 дистанціях, VO2max), ціль і `target_date` — у
`TrainingPlan`. Бракує лише чистої математики тренду і чесного вердикту.

## Дизайн

- `app/goal.py` — чистий Python-модуль у стилі `compare.py`/`wrapped.py` (нуль LLM,
  нуль мережі): лінійний тренд прогнозу цільової дистанції за останні N тижнів
  (`repository.read_history`/`get_recent_extra`-подібний рід) → проєкція на
  `target_date` → `{current_s, target_s, projected_s, slope_s_per_week, verdict}`.
  Вердикт трирівневий: on-track / близько / відстаєш-на-X.
- Бот-команда `/goal`: чистий DB-рід (`load_credentials`-патерн — без Garmin/MFA),
  детермінований текст; опційна Sonnet-нарація — лише якщо захочеться «людського»
  тону (можна почати без неї).
- Блок у тижневий дайджест: `run_digest` вже має goal + `days_to_target` — додати
  компактний `goal_projection` у контекст **і в `_digest_cache_key`** (наскрізна
  пастка README).
- План без `target_date` (open-ended `general`) → команда чесно каже «ціль без
  дати — показую тренд», без вердикту встигання.

## Acceptance criteria

- [ ] `goal.project(history, target)` — покриті тести: рівний тренд, покращення,
      регрес, недостатньо точок (→ `None`, «замало історії»), Feb-подібні дірки
      бекфілу не ламають тренд (індексація по даті, як у `correlations._paired`).
- [ ] `/goal` без активного плану → дружнє «спершу створи план на /plan».
- [ ] Race-prediction-шум: тренд по тижневих медіанах, не по сирих днях (добовий
      джитер прогнозу ~секунди — та сама причина, чому EP-14 вимагає ≥10 с margin).
- [ ] Дайджест-блок входить у `_digest_cache_key`.
- [ ] Нуль нових Claude-викликів у мінімальній версії (детермінований текст).

## Підводні камені

- Лінійна екстраполяція чесна лише на коротких горизонтах — вердикт формулювати
  як «за поточним темпом», не як обіцянку; >12 тижнів до старту → показувати тренд
  без бінарного «встигаєш».
- Garmin-прогноз ≠ реальний результат забігу — це проксі форми; у тексті так і
  називати («прогноз Garmin»), не «твій час».
- Ціль може бути «фінішувати», не часова: вердикт по часу лише якщо в intake є
  часова ціль; інакше — тренд форми (VO2max) без порівняння.

## Вартість

Нуль LLM у ядрі; опційна нарація — 1 Sonnet-виклик за запитом.
