# ST-03 · Погода в on-demand `/report` і `/report.json`

**Тип:** покращення · **Оцінка:** S · **Пріоритет:** середній (філер) · **Залежності:** немає

## Сторя

> Як користувач, я хочу, щоб `/report` у боті та `GET /report.json` враховували
> сьогоднішню погоду так само, як ранковий звіт — щоб порада про час/одяг для
> пробіжки була і на запит, а не лише зранку.

## Контекст

`fetch_forecast` (Open-Meteo, без ключа) зараз викликається тільки з morning job
(`bot/jobs.py::_fetch_user_weather` → `_deliver_morning`). `run_analysis` вже приймає
`weather=` генерично, і `weather` **вже входить у dedup-cache key** — тобто вся
механіка готова, бракує лише виклику у двох місцях.

## Acceptance criteria

- [ ] `/report` у боті передає погоду користувача (якщо задано location у `/settings`).
- [ ] `GET /report.json` — те саме.
- [ ] Користувач без location / помилка Open-Meteo → звіт працює без погоди
      (хелпер уже повертає `None` на будь-яку помилку).
- [ ] Хелпер погоди живе в одному місці — без копії між ботом і вебом.
- [ ] Ранковий шлях (`_deliver_morning`) поводиться як раніше.

## План по файлах

- `app/weather.py` — перенести сюди `_fetch_user_weather` з `bot/jobs.py:47` як
  публічний `forecast_for_user(user) -> Optional[dict]` (він лише дістає
  lat/lon користувача і кличе `fetch_forecast`).
- `bot/jobs.py` — `_deliver_morning` (~:58) використовує перенесений хелпер;
  локальний `_fetch_user_weather` видалити.
- `bot/handlers.py` — хендлер `/report`: `weather=await forecast_for_user(user)`
  у виклик `run_analysis`.
- `app/routers/reports.py` — `GET /report.json`: те саме.
- `tests/` — тест роутера: monkeypatch хелпера, перевірити прокидання параметра.

## Підводні камені

- Кеш: погода в ключі → два `/report` поспіль можуть не збігтись, якщо форекаст
  трохи змінився між викликами. Це прийнятно (так само працює ранковий шлях),
  але варто знати: часті `/report` протягом дня стануть трохи дорожчими.
- Латентність: +1 HTTP-виклик перед Claude; Open-Meteo швидкий, таймаут у
  `fetch_forecast` вже є — нічого додаткового не потрібно.
