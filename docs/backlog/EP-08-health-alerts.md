# EP-08 · Проактивні health-алерти (аномалії відновлення)

**Тип:** епік · **Оцінка:** L · **Пріоритет:** середній-високий (унікальна цінність:
годинник це знає, але мовчить) · **Залежності:** немає (дані вже збираються)

## Сторя

> Як користувач, я хочу отримати повідомлення, коли метрики тихо повзуть у поганий
> бік — HRV нижче baseline кілька днів поспіль, RHR росте, сон системно короткий,
> ACWR у червоній зоні — з поясненням і порадою (полегшити тиждень / можливо,
> захворюю) — не чекаючи, поки я сам це помічу в ранковому звіті.

## Контекст

Ранковий звіт бачить зріз «сьогодні», але ніхто не дивиться на **тренд як тригер**:
`daily_metrics.extra` вже містить resting_hr, hrv baseline band, sleep need,
readiness, acwr_pct — історія в БД, `repository.read_daily_metrics` user-scoped.
Морнінг-тік уже проходить по юзерах щодня — детектор чіпляється до нього
безкоштовно (без нового Garmin-фетчу).

## Скоуп

- **Детектор — чистий Python, без LLM**: правила на кшталт «HRV < нижньої межі
  baseline ≥3 дні», «RHR +5 уд до 7-денної медіани», «ACWR > 1.5», «сон < потреби
  ≥4 з 7 днів». Кожне правило → типізований `Alert(kind, severity, evidence)`.
- **LLM тільки для тексту**: якщо детектор щось знайшов — один Sonnet-виклик
  перетворює evidence на людське повідомлення (або навіть без LLM у v1 —
  шаблонні тексти, LLM додати потім).
- Cooldown per-rule у `bot_state` (`alert:<kind>` — не частіше раза на N днів),
  щоб не спамити тим самим трендом щодня.
- Опційний per-user вимикач у `/settings` (`alerts_enabled`, дефолт on).
- Синергія: активний алерт передається в ранковий звіт як контекст (той сам
  згадає «HRV третій день низько») — але push окремим повідомленням, бо тренд
  може зламатись і ввечері (після вечірнього синку).

## Acceptance criteria

- [ ] Правила задокументовані в коді поряд із порогами; пороги в константах.
- [ ] Жодного алерта на юзера з <7 днями історії (холодний старт).
- [ ] Cooldown працює; деактивований юзер/вимкнені алерти — тиша.
- [ ] Тести детектора на синтетичних рядах (це чиста функція — легко).
- [ ] Не медичні діагнози: формулювання «схоже на недовідновлення» + порада
      звернутись до лікаря при тривожних патернах.

## Підводні камені

- False positives вбивають довіру швидше, ніж false negatives — пороги
  консервативні, severity потрібен, і v1 краще запустити «тихо» (лог замість
  повідомлення) на 2–3 тижні для калібрування на реальних даних.
