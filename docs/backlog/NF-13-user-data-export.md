# NF-13 · Експорт власних даних (`/me/export`)

**Тип:** нова фіча (BA-аналіз 2026-07-21) · **Оцінка:** S–M · **Пріоритет:**
низький–середній (data ownership + страховка від Garmin-апокаліпсису OPS-01) ·
**Залежності:** — (чистий user-scoped DB-read); сировина для NF-08 (MCP) і
зовнішніх тулів

## Суть

У БД — роки безцінної історії (метрики+`extra`, серії пробіжок, RPE/болі з EP-12,
рекорди, плани, звіти), і вона живе на неофіційному API, який Garmin закриває.
Бекапи БД є (OPS-02), але це адмін-інструмент на Pi: користувач не може забрати
*свої* дані у відкритому форматі. Кнопка на `/me` → ZIP з JSON (+CSV для
табличного) усіх власних рядків.

## Дизайн

- `GET /me/export` (login, current user): стрімінговий ZIP —
  `daily_metrics.json` (усі колонки + розгорнутий `extra`), `activities.json`
  (+`series`, `subjective`, `analysis`), `personal_records.json`,
  `plans.json` (плани + workout'и + `steps`), `report_logs.json`
  (без чужих полів — тільки свій `user_id`; креди/токени **не** експортуються).
- CSV-дублікати для плоских таблиць (`daily_metrics.csv`, `activities.csv` без
  JSON-колонок) — для Excel/Sheets без обробки.
- Реалізація — `zipfile` у `io.BytesIO`/tempfile + `StreamingResponse`; обсяг
  особистої БД (роки × ~12 полів/день + серії по ~150 точок) — мегабайти, не гіги.

## Acceptance criteria

- [ ] Експорт містить **лише** рядки поточного користувача (тест: два юзери,
      у ZIP немає чужих id) і **жодних** секретів (креди, garth-токен,
      `password_hash` — відсутні навіть у users-зрізі; users взагалі не потрібен).
- [ ] `extra`/`series`/`steps`/`subjective` — валідний JSON у файлах, дати — ISO.
- [ ] Порожня історія → валідний ZIP з порожніми списками, не 500.
- [ ] Посилання на `/me` (кнопка «⬇️ Експорт даних»).
- [ ] Нуль Garmin- і Claude-викликів (чистий DB-read, як решта `/me`).

## Підводні камені

- Не тягнути все в пам'ять одним `all()` без огляду: activities з `series` —
  найважча таблиця; читати чанками/за роками, писати в ZIP потоково.
- `report_logs.report_text` містить повні тексти звітів — це фіча (архів
  порад), але робить файл найбільшим; окремий файл, щоб решта лишалась легкою.
- Це **не** заміна OPS-02: бекап БД лишається механізмом відновлення; експорт —
  переносимість. У доці так і зафіксувати, щоб не виникло «є ж експорт».

## Вартість

Нуль LLM.
