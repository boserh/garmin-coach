# CODE-01 · Розбити `app/analysis/service.py` (1043 рядки) на пакет

**Тип:** рефакторинг · **Оцінка:** M · **Пріоритет:** високий (файл ростиме з кожним
епіком) · **Залежності:** зробити ДО або РАЗОМ з PERF-02/PERF-04, які чіпають ті самі функції

## Проблема

Один модуль тримає шість слабо пов'язаних відповідальностей: дедуп-кеш,
пул Anthropic-клієнтів, щоденні звіти (`analyze/run_analysis`), `/ask`
(`ask_with_stats/run_ask`), аналіз активності, генерацію плану + план-едити +
генерацію силових (`_complete`, `_coerce_plan`, `generate_strength_with_stats`…).
Кожна нова фіча (адаптація, дайджест, race pack) додає сюди ще пару сотень рядків.

## Acceptance criteria

- [ ] Пакет `app/analysis/`: `cache.py` (дедуп-кеш + ключі), `client.py`
      (`_get_client`, `PRICES`, `_complete`, логування вартості), `reports.py`
      (analyze/ask/activity), `plans.py` (генерація/едити/адаптація/силові).
      `service.py` лишається тонким фасадом з реекспортами — **зовнішні імпорти
      не змінюються** (`from app.analysis import service` працює як раніше).
- [ ] Жодних поведінкових змін: тест-сьют зелений без правок тестів
      (крім patch-таргетів, якщо тести monkeypatch'ать приватні атрибути).
- [ ] Логери зберігають імена (`claude`) — формат логів не змінюється.
- [ ] `prompts.py` не чіпати (він уже окремо і цілісний).

## Підводні камені

- monkeypatch у тестах і `bot/handlers.py` може цілитись у
  `app.analysis.service.X` — реекспорт через фасад це покриває, але патчі
  "в місці використання" треба перевірити грепом.
- `_cache` — module-level стан: при рознесенні не створити другу копію
  (імпортувати один об'єкт, не копіювати ініціалізацію).
